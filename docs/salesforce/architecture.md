# アーキテクチャ設計

## システム構成

```
┌─────────────────────────────────────────┐
│         クライアント（ブラウザ）          │
│  GitHub Pages + Docsify サイト          │
└──────────────┬──────────────────────────┘
               │
               ↓
┌──────────────────────────────────────────┐
│      GitHub リポジトリ                   │
│  - Markdown ファイル                     │
│  - index.html（Docsify 設定）            │
│  - _sidebar.md（目次）                   │
└──────────────────────────────────────────┘
               │
               ↓
┌──────────────────────────────────────────┐
│    エンジニア（Git で編集）               │
│  - Markdown を書く                       │
│  - git push で自動公開                   │
└──────────────────────────────────────────┘
```

## コンポーネント

### 1. GitHub Pages
- **役割**：静的ファイルホスティング
- **配信**：index.html、Markdown ファイル
- **費用**：無料

### 2. Docsify
- **役割**：Markdown をブラウザで動的に表示
- **機能**：
  - 左サイドバー（目次）
  - 検索機能
  - レスポンシブデザイン
  - Mermaid 図式対応

### 3. Git リポジトリ
- **役割**：ソースバージョン管理
- **運用**：
  - Markdown ファイルの変更を追跡
  - 履歴管理
  - ロールバック可能

## 特徴

### 🚀 高速
- ビルドプロセスなし
- クライアント側で JavaScript により動的レンダリング

### 🔒 セキュリティ
- サーバーサイドコード不要
- スタティックファイルのみ

### 📱 モバイルフレンドリー
- レスポンシブデザイン
- タッチ操作対応

> [!WARNING]
> 本番環境では必ずリポジトリのアクセス権限を確認してください。

> [!TIP]
> ローカルプレビューには `docsify serve docs` を使用してください。
