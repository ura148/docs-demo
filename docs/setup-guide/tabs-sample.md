# タブ表示サンプル（明示マークのみ）

`docsify-tabs` は **`<!-- tabs:start -->` ～ `<!-- tabs:end -->` で囲んだブロックだけ**がタブ UI になります。囲んでいない通常の見出しや本文は、従来どおりスクロールするドキュメントのままです。

---

## 通常の見出し（タブにはならない）

次の `##` 見出しはそのまま縦に並ぶ通常のセクションです。

## セクション A

この段落は常に表示されます。

## セクション B

こちらも別タブにはなりません。

---

## 明示タブの例（見出し＋太字ラベル）

GitHub 上でもラベルが読みやすい書き方です。

<!-- tabs:start -->

#### **REST**

## エンドポイント例

`GET /api/v1/items`

レスポンス（例）:

```json
{ "items": [] }
```

#### **GraphQL**

## クエリ例

```graphql
query {
  items { id name }
}
```

#### **メモ**

同一ファイル内で、タブにしたい範囲だけを `tabs:start` / `tabs:end` で囲みます。

<!-- tabs:end -->

---

## 明示タブの例（HTML コメントでラベル）

ラベルを `<!-- tab:名前 -->` で切り替える書き方です（GitHub プレビューでは中身がそのまま見える場合があります）。

<!-- tabs:start -->

<!-- tab:開発 -->

- ベース URL: `https://dev.example.com`
- 認証: 開発用トークン

<!-- tab:本番 -->

- ベース URL: `https://api.example.com`
- 認証: 本番キー（Secrets 管理）

<!-- tabs:end -->

---

## チェックリスト

- [ ] タブにしたい範囲の**直前**に `<!-- tabs:start -->` がある
- [ ] 範囲の**直後**に `<!-- tabs:end -->` がある
- [ ] 各タブの区切りに `#### **ラベル**` または `<!-- tab:ラベル -->` がある

上記を満たさない限り、ページ内は通常の Markdown 表示のままです。
