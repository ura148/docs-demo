# プロジェクト概要

## 目的

このプロジェクトは、Markdown で書かれた技術設計書を、非エンジニアのクライアントにも見やすい形で共有することを目的としています。

## 背景

従来は Markdown → Excel に手作業で変換していましたが、以下の課題がありました：

- 変換作業が無駄
- Excel とソースの乖離が発生
- バージョン管理が困難

## ソリューション

GitHub Pages + Docsify を採用し、以下を実現：

- ✅ Markdown をそのまま公開
- ✅ Git で版管理
- ✅ 見やすいサイトをクライアント共有
- ✅ ビルドプロセス不要

## 対象ユーザー

- 技術チーム：Markdown で執筆
- クライアント：Web ブラウザで閲覧

## 期待効果

| 項目 | 従来（Excel） | Docsify |
|------|-------------|---------|
| 更新手間 | 手作業 | git push |
| バージョン管理 | 困難 | Git で自動管理 |
| 見やすさ | △ | ✅ |
| クライアント共有 | Excel ファイル配布 | URL 共有 |
| 検索機能 | なし | あり |
